REQ-22. UI 패널 React 이식 (잔여분)
  배경   : 자원 스트립·씨앗 바·토스트는 완료. 경제 순환의 나머지 절반
           (팔고, 사고, 모으는 화면)이 아직 v1에만 있다.
  내용   : 아래 패널을 React 컴포넌트로 이식하고 스토어 액션을 추가한다.
           v1 game.js의 해당 함수 로직을 그대로 옮길 것(동작 변경 금지).
           (1) 상점 패널: 씨앗 5종 목록, 구매 버튼, 도감 잠금 표시
               → 액션 buySeed(cropId)
           (2) 인벤토리 패널: 품목별 수량·개당 가격, 개별 판매, 모두 판매
               → 액션 sellItem(key), sellAll()
           (3) 오늘의 손님 패널: 요청 작물, 1.5배 안내, 납품 버튼
               → 액션 deliverVisitorOrder()
           (4) 도감 패널: 20칸 타일(??? 미발견), 보상 3종 수령 버튼
               → 액션 claimReward(rewardId)
           (5) 초기화 버튼(확인 창 포함) → 액션 resetGame()
  레이아웃: v1처럼 씬 우측(모바일에서는 아래) 패널 컬럼. v1 index.html의
           panel 구조와 styles.css를 참고해 톤 유지.
  주의   : - 1초 틱(now) 구독은 시간 표시가 필요한 컴포넌트로 한정
           - aria-live는 토스트 스택에만 유지
           - 스토어 액션은 기존 clickPlot처럼 structuredClone → 변경 →
             writeSave → set 패턴을 따를 것
  완료기준: v1에서 가능한 모든 조작(구매/판매/납품/보상 수령/초기화)이
           v2에서 동일하게 동작하고 저장에 반영된다.

           이후 스키마 변경 시 변환 함수 하나만 추가하면 되는 구조.

REQ-06. 웰컴백 요약 모달
  배경   : 방치형의 핵심 보상 체감 장치. 현재는 오프라인 성과가 토스트
           한두 줄로 흩어져 지나간다.
  내용   : (1) bootGame에서 로드 직후·보정 적용 전 상태와 비교해
               오프라인 요약을 계산: 떠나 있던 시간, 완성된 작물 수,
               시든 작물 수, 채집 리필 발생 여부, 출석 보상 여부
           (2) 오프라인 30분 이상(데모 검증 시 상수를 줄여 테스트)일 때
               모달 카드 1회 표시. 닫기 전에는 씬 조작 불가(오버레이).
           (3) 저장에는 요약을 남기지 않는다(계산 후 폐기).
  완료기준: 30분 이상 접속하지 않다가 열면 "N시간 만의 접속, 작물 M개
           완성" 형태의 요약이 뜨고, 닫으면 다시 나타나지 않는다.

REQ-08. 물주기 기본 메커니즘
  배경   : 접속해도 밭에 해줄 일이 없다. "접속 → 돌보기" 루프의 핵심.
           rollQuality(watered)가 이미 품질 확률을 올리도록 구현되어
           있으므로 트리거만 추가하면 된다.
  내용   : (1) 자라는 중인 작물을 클릭하면 (현재는 남은 시간 토스트만)
               아직 물을 안 준 작물이면 물주기로 처리: crop.watered =
               true, 물방울 튀는 간단 연출, 젖은 흙 색으로 변경(구현됨)
           (2) 이미 물 준 작물 클릭 시 기존처럼 남은 시간 토스트
           (3) 황금 물뿌리개(성장 단축)와의 관계: 황금 물뿌리개 사용 시
               watered도 함께 true(기존 동작 유지). 황금 물뿌리개
               사용 버튼을 선택된 밭 상세 카드 또는 밭 위 버튼으로 노출
  완료기준: 물 준 작물의 흙이 젖은 색으로 바뀌고, 품질 확률 상승이
           반영되며(rollQuality 인자), 작물당 1회만 가능하다.

REQ-11. 수확 연출
  배경   : 수확이 토스트 한 줄이라 코어 루프의 보상감이 약하다.
  내용   : (1) 수확 순간 작물 모델이 커졌다 사라지는 팝 애니메이션
               (useFrame 스케일 트윈 0.3초 정도면 충분)
           (2) 획득 표시: "+1 딸기 (은빛)" 플로팅 텍스트가 밭 위로
               떠오르며 사라짐 (drei Html 재사용)
           (3) 품질별 차등: 황금 품질이면 금색 파티클(작은 구 6~8개가
               퍼지는 정도), 일반/은빛은 생략 가능
  완료기준: 수확 클릭 시 3D 씬 안에서 시각 피드백이 재생되고, 연출
           중에도 다른 밭 조작이 막히지 않는다.

  내용   : (1) data.ts에 프리셋 2벌: demo(현행 값), live(기획안 값 —

           (2) URL 파라미터 ?balance=live 또는 localStorage 플래그로

  배경   : 채집물의 사용처가 판매뿐이라 채집과 밭이 연결되지 않는다.
  내용   : (1) 레시피: 작은 버섯 2 + 들꽃 1 → 비료 1 (숫자는 data.ts에)
           (2) 저장 스키마에 fertilizer 수량 추가 → version 2로 올리고
