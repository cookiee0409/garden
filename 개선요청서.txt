G-5. 웰컴백 모달·토스트·HiddenPlotStatus(접근성) 경로는 유지하고,
     새 정보(비 소식, 생물 흔적, 퇴비 완성)를 각각에 통합한다.

REQ-32. 계절과 날씨
========================================================================
목적: 접속할 때마다 "오늘 정원은 어떤 모습일까"가 달라지게 한다.

  (1) 계절 판정 (logic.ts, 순수 함수)
      getSeason(now): 3~5월 spring / 6~8 summer / 9~11 autumn /
      12~2 winter. 실제 날짜 기준, 저장 불필요.
  (2) 날씨 판정 (logic.ts, 순수 함수 — 결정적)
      getWeather(now): hashString(getDayKey(now) + ":weather") 기반으로
      하루 단위 고정. 맑음 75% / 비 25% (겨울은 비 대신 눈).
      난수를 새로 뽑지 말 것 — 같은 날은 항상 같은 날씨여야 한다.
  (3) 씬 연출 (GardenScene)
      - 계절별 팔레트: 섬 잔디·나무 색, 캔버스 배경색을 계절×밤낮
        조합으로 전환 (기존 night 전환과 같은 방식, 상수는 layout.ts
        또는 별도 palette.ts에)
      - 파티클: 봄 벚꽃잎 / 여름 빗줄기 / 가을 낙엽 / 겨울 눈.
        인스턴스 최대 40개, useFrame 재사용 버퍼로 구현(G-3).
        비/눈은 비 오는 날만, 벚꽃·낙엽은 항상 은은하게.
  (4) 비 오는 날 효과
      - 그날 처음 접속(또는 자정 경과) 시 자라는 중인 모든 작물을
        watered 처리하고 lastRainWateredDate = 오늘로 기록.
        토스트 "비가 내려 밭이 촉촉해졌습니다."
      - 그날 심는 작물도 심는 즉시 watered 처리.
      - 웰컴백 모달에 "비 소식" 줄 추가.
  (5) 계절 한정 채집물 4종 추가 (FORAGE_DEFS에 season 필드)
      예: 봄 벚꽃잎, 여름 매미 허물, 가을 도토리, 겨울 눈꽃 조각.
      해당 계절에만 채집 테이블에 등장(가중치 낮게). 도감 +4칸.
  완료기준:
      a. 시스템 날짜를 바꾸면 계절 팔레트·파티클이 바뀐다
      b. 같은 날짜는 새로고침해도 날씨가 동일하다
      c. 비 오는 날 첫 접속 시 자라는 작물이 전부 물줌 상태가 된다
      d. 계절 채집물이 해당 계절에만 나오고 도감에 등록된다
      e. v2 저장을 열어도 오류 없이 v3로 마이그레이션된다

REQ-33. 장식품과 아늑함
========================================================================
목적: 정원을 내 취향으로 꾸미고, 꾸밈이 게임에도 좋은 일이 되게 한다.

  (1) 데이터 (data.ts): DECOR_DEFS 5종
      울타리(60G, cozy 2) / 화분(90G, cozy 3) / 벤치(160G, cozy 5) /
      가로등(220G, cozy 6, 밤 점등) / 미니 연못(320G, cozy 8)
      각 항목: id, name, cost, cozy, colliderRadius.
  (2) 구매: 상점 패널에 "장식" 섹션 추가. 구매 시 보유 장식 목록
      (미배치 상태)으로 들어간다.
  (3) 배치 모드
      - 보유 장식 선택 → 배치 모드 진입: 캐릭터 앞 0.8유닛 지점에
        반투명 미리보기가 따라다니고, 유효/무효를 색으로 표시
        (유효 = 걷기 가능 영역이며 밭·포털·표지판·다른 장식·나무와
        겹치지 않고 섬 걷기 한계 안)
      - E(또는 터치 버튼)로 설치, ESC/취소 버튼으로 배치 모드 종료
      - 설치된 장식은 충돌체가 된다 (colliderRadius,
        layout의 충돌 체계에 동적으로 합류)
      - 회수: 장식에 근접하면 하단 패널에 "회수" 버튼 → 보유 목록으로
  (4) 아늑함(coziness)
      - 파생값: 배치된 장식 cozy 합계. 자원 스트립에 표시.
      - 효과 1: 손님 정산 배율 1.5 → 1.5 + min(0.3, coziness * 0.01)
        (예: cozy 20이면 1.7배). 손님 패널에 현재 배율 표기.
      - 효과 2: REQ-34 생물 방문 확률 가중치로 사용.
  (5) 저장: decorations: { id, type, x, z, rotY }[] — v3 필드(G-1).
  완료기준:
      a. 구매→배치→회수가 걷기·근접 상호작용으로 동작한다
      b. 무효 위치(밭 위, 포털 위, 장식 겹침)에는 설치되지 않는다
      c. 새로고침 후 장식 위치·아늑함이 유지된다
      d. 가로등이 밤에만 점등한다 (pointLight는 가로등당 1개,
         최대 동시 점등 4개로 제한 — 성능)
      e. 손님 배율에 아늑함이 반영되고 UI에 보인다

REQ-34. 정원 방문 생물과 생물 도감
========================================================================
목적: 정원이 나 말고도 누군가 찾아오는 살아있는 공간이 되게 한다.

  (1) 데이터 (data.ts): CRITTER_DEFS 6종 — 등장 조건과 함께
      나비   : 낮 + 꽃 계열 작물(해바라기·무지개꽃) 재배 중
      참새   : 낮 + 물 준 밭 2칸 이상
      토끼   : 낮 + 당근 재배 중
      개구리 : 비 오는 날
      고슴도치: 밤 + 아늑함 5 이상
      부엉이 : 밤 + 나무 근처 (등장 위치가 나무 옆)
  (2) 등장 로직
      - 접속 중 90~180초(데모)마다 판정: 조건 충족 생물 중 무작위 1종,
        기본 확률 35% + 아늑함×1% (상한 65%). 최대 동시 2마리.
      - 등장한 생물은 30~60초 머물다 떠난다. 섬 위를 느리게 배회
        (위치는 ref/useFrame — G-3, 충돌 무시하고 밭 위는 피해서 배회)
      - 절차적 미니 모델 (CritterModel.tsx 신규, 종당 도형 3~5개)
  (3) 관찰 상호작용
      - 생물이 근접 대상에 포함됨(반경 1.1, 프롬프트 "E — 관찰").
      - 관찰해도 도망가지 않는다. 하트 이펙트 + 첫 관찰이면 생물 도감
        등록("critter|{id}|normal" 키로 기존 codex 맵 재사용) +
        골드 30. 이미 등록된 생물은 하트만.
  (4) 방치 흔적
      - 오프라인 6시간(데모 10분) 이상 후 접속 시, 그 시간대 조건을
        충족했던 생물 1종의 흔적을 웰컴백 모달에 표시
        ("밤사이 고슴도치가 다녀간 흔적이 있어요"). 도감 등록은 안 됨
        (직접 봐야 등록 — 접속 유인 유지).
  (5) 도감 패널에 "생물" 섹션 추가 (??? 미발견 타일 동일 패턴).
  완료기준:
      a. 조건을 맞추면(예: 해바라기 재배+낮) 몇 분 안에 나비가 나타난다
      b. E 관찰로 도감 등록·골드 지급, 재관찰은 하트만
      c. 동시 2마리 초과 등장하지 않고, 프레임 저하가 없다
      d. 오프라인 후 웰컴백에 흔적 문구가 뜬다
      e. 도감 카운트가 36칸 기준으로 표시되고 G-4 신규 보상이 동작한다

REQ-35. 퇴비함 — 시든 작물의 순환
========================================================================
목적: 늦게 접속한 벌(시듦)을 "완전한 손해"에서 "다른 자원"으로 바꾼다.

  (1) 오브젝트: 정원 고정 위치(스폰 근처 가장자리, layout.ts에 좌표)에
      퇴비함 1개를 기본 제공. 절차적 모델(나무 상자 + 흙 더미).
      충돌체이자 근접 상호작용 대상.
  (2) 규칙
      - 슬롯 2칸. 근접 후 "시든 작물 넣기" → 인벤토리의 wilted 품목을
        오래된 것부터 1개 소모, 슬롯에 {itemKey, startedAt} 기록
      - 완성 시간: 데모 3분 / 실서비스 6시간 (BALANCE_PRESETS)
      - 완성된 슬롯은 근접 E로 수거 → 비료 +1
      - 오프라인 경과도 startedAt 기준으로 자연 반영(기존 시간 패턴)
  (3) UI
      - 근접 시 하단 패널: 슬롯별 상태(비어 있음/N분 남음/완성)와
        넣기·수거 버튼. 프롬프트 문맥 행동: 완성 슬롯 있으면 "수거",
        아니면 시든 작물 보유 시 "넣기".
      - 퇴비함 위 Html 라벨: 진행 중이면 남은 시간, 완성이면 "완성!"
      - 웰컴백 모달에 "퇴비 완성 N개" 줄 추가(G-5).
  (4) 저장: compost: { slots: ({ itemKey, startedAt } | null)[] }
      — v3 필드(G-1).
  완료기준:
      a. 시든 작물 넣기 → 대기 → 수거 → 비료 증가 사이클 동작
      b. 넣은 채 종료 후 완성 시간 경과 뒤 재접속하면 완성 상태
      c. 시든 작물이 없으면 넣기 버튼 비활성 + 안내
      d. 웰컴백 요약에 퇴비 완성이 집계된다
