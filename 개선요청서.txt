0. FIX-02. 정원에서 캐릭터가 대부분의 밭에 도달할 수 없음  [긴급]
========================================================================

■ 증상
  스폰 지점(남쪽)에서 남쪽 밭 3칸과 포털만 접근 가능하고, 나머지
  밭(특히 중앙 밭)으로는 걸어갈 수 없다. 겉보기에는 나무가 길을
  막는 것으로 보인다.

■ 원인 분석 (좌표 수치 — GardenScene.tsx 기준)
  네 가지 수치가 겹쳐서 생긴 문제다. 나무만 옮겨서는 해결되지 않는다.
  (1) 밭 사이 틈이 캐릭터보다 좁다.
      밭 간격 1.3 - 타일 폭 1.1 = 틈 0.2 < 캐릭터 지름 0.48
      → 3×3 그리드 내부 통로가 전부 막혀, 그리드 전체가 한 덩어리
      장애물(|x|,|z| ≤ 1.85+0.24 = 2.09)이 된다.
  (2) 섬이 그리드 모서리보다 작다.
      걷기 한계 반경 = ISLAND_RADIUS 3.05 - 0.24 = 2.81 인데
      그리드 모서리 대각 거리 = √(2.09²+2.09²) = 2.96 > 2.81
      → 그리드 바깥 외곽 회랑이 네 모서리에서 완전히 닫힌다.
      (각 변의 회랑은 |x| < 1.88 구간만 통행 가능한 고립 구간)
  (3) 나무가 회랑을 정통으로 막는다.
      정원 나무 (2.55, 1.55)와 (-2.55, -1.75)의 유효 차단 반경
      0.28+0.24 = 0.52가 동/서 회랑(폭 2.09~2.81)을 전부 덮는다.
  (4) 중앙 밭은 상호작용 자체가 불가능하다.
      중앙 밭(0,0)까지의 최근접 접근 거리 2.09 > 상호작용 반경 0.9.

■ 수정 방법 (권장 수치 — 시각 균형에 맞춰 미세 조정 가능하나,
  아래 완료기준을 전부 만족해야 한다)
  (1) 밭 간격 1.3 → 1.7  (십자 통로 폭 0.6 확보, 캐릭터 0.48 통행)
  (2) 섬 확장: ISLAND_RADIUS 3.05 → 4.3
      정원 섬 메시 반지름 3.4/3.8 → 4.45/4.85 (숲 섬도 동일 비율 확장,
      두 씬의 스케일 톤을 맞출 것)
  (3) 나무는 걷기 한계(4.06) 바깥, 반경 4.15~4.35 링에만 배치.
      회랑 어느 지점에서도 통행 폭 0.6 이상 보장.
      숲의 나무 4그루도 같은 기준으로 재배치.
  (4) 포털·스폰 재배치 (섬 확장 반영 + 아래 IMP-01 동시 해결):
      정원 포털 z 2.72 → 3.9, 정원 스폰 z 2.12 → 2.9
      숲 포털 z -2.72 → -3.9, 숲 스폰 z -2.12 → -2.9
      (스폰-포털 거리 1.0 > 상호작용 반경 0.9 → 도착 직후 포털
      프롬프트가 뜨지 않게 됨)
  (5) 숲 채집 스팟 산포 계수 (5.6, 4.8) → (6.6, 5.6) 정도로 확대해
      커진 섬에 맞게 퍼뜨릴 것 (모든 스팟이 걷기 한계 안에 있어야 함).
  (6) CAMERA_OFFSET (5.2, 6.2, 5.2) → 약 (6.6, 7.9, 6.6)으로 늘려
      확장된 섬과 회랑이 화면에 들어오게 조정.
  (7) [재발 방지] 위 수치들(밭 간격, 섬 반경, 나무·포털·스폰 좌표,
      상호작용 반경)이 GardenScene.tsx 안에 흩어져 있다.
      v2/src/scene/layout.ts (신규)로 전부 모으고 GardenScene은
      import해서 쓸 것. 앞으로 배치 조정은 이 파일만 수정.

■ 완료기준 (전부 실제 조작으로 확인 후 커밋)
  a. 스폰에서 9개 밭 전부 — 중앙 밭 포함 — 걸어가서 E로 상호작용 가능
  b. 밭 사이 십자 통로를 캐릭터가 걸림 없이 통과
  c. 정원 외곽을 한 바퀴(모서리 4곳 포함) 걸어서 돌 수 있음
  d. 포털 왕복 가능, 씬 도착 직후 포털 프롬프트가 자동으로 떠 있지 않음
  e. 숲의 채집 스팟 6곳 전부 도달·채집 가능
  f. npx tsc -b 와 npm run build 통과
  커밋: "fix: FIX-02 ..." (layout.ts 분리 포함)

IMP-02. 프레임당 불필요한 객체 생성 절감  [폴리시]
  위치   : GardenScene.tsx PlayerController.useFrame,
           nearestInteraction()
  내용   : (1) nearestInteraction이 매 프레임 targets 배열과 객체를
           새로 만든다 — 거리 검사를 먼저 하고 프롬프트 생성은 최근접
           대상이 있을 때만 하도록 이미 되어 있으나, targets 배열
           생성 자체를 씬/스팟 변경 시에만 갱신되는 캐시로 바꿀 것.
           (2) 카메라 추적의 cameraTarget.clone().add(...)를 재사용
           Vector3로 교체 (매 프레임 2회 할당 제거).
  완료기준: 동작 변화 없음. 코드상 매 프레임 new/clone/배열 리터럴이
           제거됨.
