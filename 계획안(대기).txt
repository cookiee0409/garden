대기 1. 반려 동물 (호감도 시스템)                       예상 규모: 중
------------------------------------------------------------------------
개념   : 오늘의 손님(루나, 솔, 마루, 노아, 아라)의 요청을 여러 번
         들어주면 호감도가 쌓이고, 일정 단계에 도달하면 그 손님의
         동물 친구가 정원에 정착한다.
시스템 초안:
  - 손님별 호감도: 납품 1회 +1, 5/10/15 단계 보상
    (씨앗 → 장식품 → 펫 정착)
  - 펫은 상주 생물(REQ-34 생물과 달리 떠나지 않음), 하루 1회
    자동 보조 행동: 물 안 준 밭 1칸에 물주기, 또는 채집물 1개 주워다
    주기 (펫 종류별로 다르게)
  - 펫 하우스(장식품 연계), 쓰다듬기 상호작용(E)
저장   : visitorAffinity: Record<이름, number>, pets: string[]
         → 저장 버전 +1 필요
선행조건: REQ-34(생물 모델·배회 로직 재사용), REQ-33(장식 연계)
비고   : 손님 시스템이 "일일 거래"에서 "관계"로 깊어지는 확장.
         힐링 게임의 정서적 핵심이 될 수 있어 우선 검토 권장.

------------------------------------------------------------------------
대기 2. 정원 설비 자동화                                예상 규모: 중
------------------------------------------------------------------------
개념   : 골드의 후반 사용처. 성장할수록 손이 덜 가는 방치형 곡선.
시스템 초안:
  - 스프링클러(800G): 설치 위치 기준 인접 밭 최대 4칸,
    하루 1회 자동 물주기
  - 허수아비(500G): 시듦 유예 시간 2배 (WILT_AFTER_MS × 2)
  - 온실(1,500G): 전용 밭 1칸 추가, 계절 무관·시듦 없음,
    성장 시간 +25% 페널티
  - 설치는 장식품 배치 모드(REQ-33) 재사용, 설비는 아늑함 대신
    기능 효과를 가짐
저장   : decorations에 kind: "decor" | "utility" 필드 추가로 통합 가능
선행조건: REQ-33 배치 시스템
비고   : 실서비스 밸런스(live)로 지인 테스트를 시작한 뒤, 골드가
         남아도는 시점이 관측되면 착수하는 것이 좋다.

------------------------------------------------------------------------
대기 3. 연못과 낚시 (세 번째 씬)                        예상 규모: 중~대
------------------------------------------------------------------------
개념   : 잔잔히 앉아 기다리는 힐링 콘텐츠 + 물고기 도감.
시스템 초안:
  - 숲 입구에서 이어지는 세 번째 섬 "연못가" (포털 체인:
    정원 ↔ 숲 ↔ 연못)
  - 낚시터에 근접 E → 낚시 시작 → 3~10초 무작위 대기 → 찌 흔들림
    연출 → 1.2초 안에 E를 다시 누르면 성공
  - 물고기 6종: 시간대(낮/밤)·날씨(비 오는 날 확률 상승)·계절 한정
    1종. 판매가는 채집물보다 약간 높게. 도감 +6칸
  - 하루 낚시 횟수 제한(미끼 5개, 자정 리필)으로 방치 리듬 유지
저장   : bait: number, 물고기는 기존 인벤토리·도감 체계 재사용
선행조건: REQ-32(날씨·계절 연동 시 재미가 배가됨)
비고   : 씬 하나가 통째로 늘어나므로 이 목록에서 가장 크다.
         모바일 성능(씬 전환 시 메모리) 점검 필요.

------------------------------------------------------------------------
대기 4. 꽃 교배                                         예상 규모: 대
------------------------------------------------------------------------
개념   : 이웃한 두 밭의 꽃이 확률로 새 색상 변종 씨앗을 만든다.
         수집 엔드 콘텐츠.
시스템 초안:
  - 해바라기·무지개꽃(+추후 꽃 작물)이 서로 인접 밭에서 동시에
    완성 상태로 존재하면, 수확 시 확률(10~20%)로 변종 씨앗 획득
  - 변종: 색상 팔레트 교체로 모델 재사용 (예: 붉은 해바라기,
    달빛 해바라기) — 종당 변종 2~3개, 도감 대폭 확장
  - 변종은 판매가가 높고 일부는 특정 조합·밤·비 조건 필요
주의   : 도감 칸 수가 크게 늘어 보상 요구치 전면 재조정 필요.
         "인접" 판정과 확률 설계가 밸런스에 민감 — 실서비스 밸런스
         검증(지인 테스트) 후 착수할 것.
선행조건: REQ-32, 밸런스 안정화
비고   : 이 목록에서 가장 마지막에 할 것.
