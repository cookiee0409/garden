# 🌱 미니 방치형 정원 게임 — 기획안

> 작성일: 2026-07-03 · 버전 0.1
> 전제: 1인 개발(프로그래밍 초보, Claude Code와 협업), 웹게임, 싱글 플레이 우선

---

## 1. 게임 콘셉트

**한 줄 요약**: 하루에 한두 번, 5~10분씩 들러서 나만의 아기자기한 3D 정원을 가꾸는 힐링 방치형 게임.

- **장르**: 방치형(idle) + 농장 경영 + 채집
- **플랫폼**: 웹 브라우저 (PC/모바일 반응형)
- **플레이 세션**: 1회 5~10분, 하루 1~2회 접속이 최적이 되도록 설계
- **감성 키워드**: 아기자기, 파스텔톤, 말랑말랑한 로우폴리 3D, 힐링

**핵심 재미**:
1. 심고 → 기다리고 → 수확하는 성취감 (방치 보상)
2. 정원이 점점 넓어지고 예뻐지는 시각적 성장
3. 도감을 채우는 수집욕 (희귀 식물, 채집물)

---

## 2. 핵심 게임 루프

```
씨앗 구매 → 밭에 심기 → (시간 경과: 접속 안 해도 자람) → 수확
    ↑                                                      ↓
  돈으로 씨앗/밭/도구 구매 ← 상점에 판매 ← 인벤토리에 보관
```

보조 루프:
- **채집**: 정원 주변(숲 입구)에서 하루 몇 번 리필되는 채집 포인트에서 버섯·열매·꽃 줍기
- **도감**: 수확/채집한 모든 것이 도감에 등록. 도감 달성 보상(씨앗, 장식품)
- **정원 확장**: 돈으로 밭 칸 늘리기, 장식품 배치 (꾸미기 요소)

---

## 3. 하루 1~2회 접속 유인책 (핵심 설계)

방치형의 성패는 "다시 접속할 이유"에 달려 있습니다. 아래 장치를 조합합니다.

| 장치 | 설명 | 접속 주기 유도 |
|---|---|---|
| **작물 성장 시간** | 작물별 성장 시간을 4~12시간대로 설계. 아침에 심으면 저녁에 수확 | 하루 2회 |
| **채집 포인트 리필** | 채집 스팟이 6시간마다 리필 (최대 2회분까지만 누적) | 하루 1~2회 |
| **방치 보상 상한** | 오프라인 성장 계산은 최대 12시간까지만 적용 → "너무 오래 안 오면 손해" | 최소 하루 1회 |
| **시든 작물** | 다 자란 후 24시간 방치하면 시들어 판매가 하락 (소멸은 아님 — 스트레스 최소화) | 수확 타이밍 의식 |
| **일일 손님** | 하루 한 번 정원에 손님(고슴도치, 새 등)이 방문해 특정 작물을 웃돈 주고 요청 | 하루 1회 |
| **연속 출석 물뿌리개** | 매일 첫 접속 시 "황금 물뿌리개" 1회 — 원하는 작물 성장 시간 단축 | 하루 1회 |

**설계 원칙**: 벌주기(소멸, 도둑)보다 보상 놓침(리필 상한, 손님 요청) 위주로.
힐링 게임이므로 "안 들어오면 망한다"가 아니라 "들어오면 좋은 일이 있다"로 느껴져야 함.

## 4. 주요 시스템 상세

### 4.1 밭 & 작물
- 시작: 3×3 밭 중 4칸 개방. 돈으로 칸 확장 (최대 5×5 → 추후 두 번째 정원)
- 작물 티어 예시:

| 티어 | 예시 | 성장 시간 | 씨앗값 → 판매가 | 특징 |
|---|---|---|---|---|
| 1 | 방울토마토, 당근 | 30분~1시간 | 10 → 25 | 첫날 튜토리얼용, 빠른 보상 |
| 2 | 딸기, 해바라기 | 4~6시간 | 50 → 140 | 하루 2회 접속 리듬의 중심 |
| 3 | 수박, 장미 | 10~12시간 | 200 → 600 | 자기 전에 심는 용도 |
| 4 | 달빛버섯, 무지개꽃 | 24시간+ | 1,000 → 3,500 | 후반 콘텐츠, 희귀 씨앗 필요 |

- **품질 시스템**: 수확 시 일반/은빛/황금 품질 랜덤 (물주기·비료로 확률 상승) → 같은 작물도 반복할 이유

### 4.2 채집
- 정원 옆 "숲 입구" 씬에 채집 스팟 5~8개 (반짝이는 오브젝트 탭/클릭)
- 6시간마다 리필, 계절/시간대별로 나오는 아이템 차등 (밤에만 나오는 반딧불이 등)
- 채집물은 판매 or 비료 제작 재료

### 4.3 경제
- 화폐는 **골드 하나**로 시작 (단순하게). 프리미엄 화폐 없음 — 수익화는 나중 고민
- 돈 쓸 곳: 씨앗 → 밭 확장 → 도구 업그레이드(물뿌리개, 비료통) → 장식품(순수 꾸미기)
- 인플레 방지: 상위 티어 씨앗은 골드 + "도감 진행도" 잠금 해제 조건 병행

### 4.4 도감 & 수집
- 작물 × 품질, 채집물 전부 도감 등록
- 구간 달성 보상: 희귀 씨앗, 장식품, 밭 확장 할인권
- 장기 목표 역할 (엔드 콘텐츠)

## 5. 아트 디렉션

- **스타일**: 로우폴리(low-poly) + 파스텔 색상 + 부드러운 그림자. 참고: 《동물의 숲》 미니어처 감성, 게임 《Alba》, 《Tiny Glade》
- **에셋 조달 전략** (초보 1인 개발 기준, 직접 모델링 최소화):
  1. **Kenney.nl** — 무료 로우폴리 3D 에셋 (식물, 자연물 팩 있음, CC0 라이선스)
  2. **Quaternius** — 무료 로우폴리 팩 (Ultimate Nature Pack 등)
  3. 부족한 것만 Blender로 간단 제작 or 유료 에셋 구매
- **연출 포인트**: 수확 시 뿅 하는 스케일 애니메이션 + 파티클, 작물 성장 단계 3단계(새싹→중간→완성) 모델 교체, 낮/밤 조명 변화

## 6. 기술 스택

초보자 기준으로 "왜 이걸 쓰는지"까지 설명합니다.

| 영역 | 선택 | 이유 |
|---|---|---|
| 언어 | **TypeScript** | 실수를 코드 작성 시점에 잡아줘서 초보에게 오히려 유리 |
| 프레임워크 | **React + Vite** | 인벤토리·상점 같은 UI를 만들기 쉬움. Vite는 개발 서버가 빠르고 설정이 거의 없음 |
| 3D | **React Three Fiber (R3F) + drei** | Three.js를 React 방식으로 쓰게 해주는 라이브러리. drei는 자주 쓰는 기능(카메라 컨트롤, 로더) 모음 |
| 상태 관리 | **Zustand** | 게임 데이터(골드, 인벤토리)를 전역으로 관리. R3F와 같은 개발팀이라 궁합이 좋음 |
| 백엔드 | **Supabase** | 계정(구글 로그인), DB, 저장을 코드 거의 없이 제공. 무료 티어로 충분 |
| 배포 | **Vercel** | GitHub에 올리면 자동 배포. 무료 |

### 오프라인 성장 계산 (방치형의 핵심 원리)
서버가 실시간으로 작물을 키우는 게 아니라, **심은 시각(타임스탬프)만 저장**해 두고
접속 시 `현재 시각 - 심은 시각`으로 성장 상태를 계산합니다.
- 치팅 방지를 위해 시간 기준은 클라이언트가 아닌 **서버 시각(Supabase)** 사용
- 이 구조 덕분에 서버에 상주 프로세스가 필요 없음 → 운영비 0원 유지 가능

### 데이터 모델 초안 (Supabase 테이블)
```
players   : id, gold, last_login, streak(연속출석), created_at
plots     : id, player_id, position, unlocked(개방 여부)
crops     : id, plot_id, crop_type, planted_at, watered, quality_bonus
inventory : id, player_id, item_type, quantity
codex     : id, player_id, item_type, quality, first_obtained_at
```

## 7. 개발 로드맵

한 번에 다 만들지 않고, **매 단계가 "돌아가는 게임"**이 되도록 쪼갭니다.

### Phase 0 — 준비 (1주)
- Vite + React + TypeScript + R3F 프로젝트 셋업
- Kenney/Quaternius 에셋 받아서 화면에 3D 밭 하나 띄우기
- 목표: "브라우저에 내 정원이 보인다"

### Phase 1 — 코어 루프 MVP (2~3주) ⭐ 가장 중요
- 3×3 밭, 작물 2종, 심기/수확, 골드, 상점(씨앗 구매/작물 판매)
- 저장은 일단 localStorage (Supabase는 다음 단계)
- **목표: 심고→기다리고→수확하고→판다가 완전히 동작**
- 이 단계가 재미없으면 그 위에 뭘 쌓아도 재미없음 → 여기서 충분히 다듬기

### Phase 2 — 방치 & 계정 (2주)
- Supabase 연동: 구글 로그인, 서버 저장, 서버 시각 기준 오프라인 성장
- 방치 보상 상한(12시간), 시듦 시스템
- 목표: "자고 일어나면 자라 있다"

### Phase 3 — 재접속 유인책 (2주)
- 채집 씬 + 리필 시스템, 일일 손님, 연속 출석 물뿌리개
- 작물 티어 4단계 확장, 품질 시스템
- 목표: "내일 또 들어오고 싶다"

### Phase 4 — 수집 & 꾸미기 (2~3주)
- 도감 + 달성 보상, 밭 확장, 장식품 배치
- 낮/밤 조명, 수확 연출, 사운드
- 목표: "오래 하고 싶고, 남에게 보여주고 싶다"

### Phase 5 — 공개 & 다듬기
- Vercel 배포, 지인 테스트, 밸런스 조정 (성장 시간·가격표는 반드시 실플레이 후 수정)
- 이후 선택지: 가벼운 소셜(친구 정원 방문), 계절 이벤트, 모바일 PWA

> 총 기간 감각: 주말 위주 작업 기준 3~4개월. Phase 1까지만 가도 게임의 성패를 판단할 수 있습니다.

## 8. 리스크와 대응

| 리스크 | 대응 |
|---|---|
| 3D 학습 곡선이 가파름 | Phase 0~1에서는 카메라 고정(쿼터뷰) + 클릭만. 회전/이동 컨트롤은 나중에 |
| 에셋 퀄리티 들쭉날쭉 | 처음부터 Kenney 또는 Quaternius **한 곳 팩으로 통일**해 톤 유지 |
| 밸런스 감이 없음 | 성장 시간·가격을 코드에 하드코딩하지 말고 **데이터 파일(JSON) 하나로 분리** → 숫자만 고쳐서 실험 |
| 스코프 폭발 | 소셜·이벤트·수익화는 전부 Phase 5 이후로. 이 문서에 없는 기능은 일단 안 만듦 |
| 모바일 성능 | 로우폴리 유지, 그림자 저해상도, 밭 최대 크기 제한 |
